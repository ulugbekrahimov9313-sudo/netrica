# 5. XAVFSIZLIK TALABLARI


## 5.1. Autentifikatsiya va Avtorizatsiya Xavfsizligi


## 5.2. Ma'lumotlar Xavfsizligi
- HTTPS majburiy — barcha aloqa SSL/TLS orqali shifrlangan
- SQL injection oldini olish — ORM va parametrlangan so'rovlar ishlatish
- XSS himoya — barcha foydalanuvchi kiritmalari sanitizatsiya qilinadi
- CSRF himoya — CSRF token barcha POST/PUT/DELETE so'rovlar uchun
- API kalitlarni muhit o'zgaruvchilarida saqlash — kod ichida hech qachon yo'q
- Maxfiy ma'lumotlarni loglarda ko'rsatmaslik
- Database backup — kunlik avtomatik zaxira


## 5.3. API Xavfsizligi
- Rate limiting: IP bo'yicha daqiqada 60 so'rov, soatda 1000 so'rov
- Request size limit: 10MB dan oshmasin
- CORS — faqat ruxsat etilgan domenlardan so'rovlarni qabul qilish
- API versiyalash — kelajakdagi o'zgarishlar orqaga mos bo'lishi
- Input validation — barcha kiruvchi ma'lumotlar Pydantic orqali tekshiriladi


## 5.4. Fayl Xavfsizligi
- Foydalanuvchi yuklagan fayllar turi va hajmini tekshirish
- Ruxsat etilgan fayl turlari: PDF, DOCX, JPG, PNG, (maks 10MB)
- Fayl nomlari tasodifiy UUID bilan qayta nomlanadi
- Fayllar to'g'ridan-to'g'ri webdan kirish bo'lmagan papkada saqlanadi
